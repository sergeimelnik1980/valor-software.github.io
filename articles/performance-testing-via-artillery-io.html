Performance Testing via Artillery.io

Dmitriy Zhemchugov

QA Engineer
January 18, 2022
Traduzido por

Today, I’m going to tell you all about setting up and using Artillery.io on your projects. Artillery.io is modern performance testing software that is incredibly powerful, but still very simple to navigate. ‌

The software is a Node.js tool, with a basic СLI. Being a Node.js tool makes installing Artillery.io incredibly easy through the npm utility. What’s even better? – There’s a free ‘Dev’ option if you want to test the waters before diving in.

Why Artillery.io?

Artillery.io provides many services that make performance and load testing extremely faster and more reliable. There are many reasons why I choose to use Artillery over the better known software, and I’ll outline a few of them for you.

Node.js tool

Java-based tools can be complicated and time-consuming to work with. Artillery.io is a Node.js tool, which is a far simpler way to run performance tests.

Broad language support

Out of the box, Artillery.io will support:

  • HTTP
  • Socket.io
  • WebSocket

However, Artillery.io can be used to test any backend, thanks to the plugin interface. A few examples of the protocol plugins are:

  • Kafka
  • Kinesis
  • HLS‌

YAML format

Unlike popular competitors’ scalable testing tools, Artillery.io writes tests in YAML format. This makes the tests comfortably readable and easy to understand. The performance testing software also supports a JSON format for testing.

Pre-production load performance testing

Artillery.io makes it easy to run large-scale load tests in preparation for heavy site traffic because it works in any language or protocol. As a result, you are able to use scalability performance testing to fix problems before they happen.

Accessible scenarios

The scenarios and phases from Artillery.io make the tests that you run more flexible, which enables you to check multiple use cases at the same time. Even more convenient, it’s easy to reuse scenarios for comparison and future testing.

Easy integration

Artillery.io seamlessly integrates with CI (for example, check the integration guide for GitHub Actions), making the process very organized. Artillery also works with Docker, so you can be sure that libraries will be in appropriate channels for simple communication.

Detailed reports

After running tests, create JSON reports through the --output flag. After that, you can use the JSON report to create an HTML report to be saved as a file or exported to your default browser through the --output flag once more.

Artillery Docs

On the Artillery website, there are numerous documents to browse for instructions and other information users may require. This makes getting the performance testing software up and running much smoother than with previous tools. The community forums allow easy contact with other developers.

How does Artillery.io work?

I’m going to walk you through some of the basic commands to get you started with Artillery.io. Valor Software, the creator of many Angular community libraries, including ngx-bootstrap, has hundreds of thousands of developers and engineers using their products monthly, and so needs scalable performance testing to make sure everything runs smoothly. We’ll use the ngx-bootstrap site in the following examples.

First, you must install the latest version of the software.

Install

To install the latest version, enter:


npm install -g artillery@latest

Fast test

To run a fast test, enter:


artillery quick --count 15 -n 30 https://valor-software.com/ngx-bootstrap/

With this command, the test will run 15 virtual users with each of them making 30 HTTP GET requests to the URL address that you specified.‌

Config

For standard test scenarios, you should create a ‘my.yml’ file with config.

The my.yml file should contain ‘config:’ paired with the target URL. To set up environments for your tests, you can write environments into the command after the target URL has been entered.

Phases in the config

Artillery.io allows you to enter multiple sequential load options for the application being tested. The testing phase consists of several aspects:

  • Duration: the time of one phase
  • arrivalsRate: the number of users added each second
  • ramptTo: up to how many users per second the load will grow by
  • name: a name of the phases

If you only have one target url, then the different phases of the performance load testing are placed directly after it. If there are multiple environments, then the phases should be added to each environment variable.

Plugins

On the Artillery.io website, there are lots of plugins that can help you in your performance testing.

Install artillery-plugin-expect to compare the expected result with the actually received result.

Then, after the phases inside the config, enter:


plugins:
   expect: {}

Now, with the config completed, we can run some tests!

Scenario

All tests should be written in the ‘scenarios’ section, and should contain:

  • GET, POST, PUT, DELETE, and some other commands
  • A URL for every endpoint
  • The body text in YML format
  • Ensure all necessary checks have been done

To run tests, you should enter:


artillery run my.yml

To run a test and then generate a report in a .txt file, run:


artillery run -o first.txt my.yml

After generating a .txt report, to generate a second report in HTML, run the command:


artillery report my.txt

Authentication

With Artillery.io, you can use basic authentication. You get authorized by tokens, and can upload CSV with your credentials whenever you want.

Check their official documentation to find information on authentication and many other aspects of the Artillery.io software.

For example, to upload a CSV file with credentials, you should add the following lines into the config file:

Payload – to use payload functionality

Path – to write a path for the CSV file that Artillery needs to use

Fields – names of fields that you need to use

Final thoughts

Artillery.io is an incredibly accessible and hard-working tool, and I highly recommend it for performance testing.

Artillery Core is the free version of the software, where the test can be run from a local or virtual machine. There is then the smooth upgrade to Artillery Pro, where you will have a self-hosted AWS performance testing platform.

Other features of Artillery Pro include:

  • Runs millions of tests per second across 13 geographical regions
  • Works with existing security systems
  • No repeat charges or paid maintenance
  • VPC internal test services

I hope this article has given you some clarity on Artillery’s software and eased your choice of the performance testing tool.‌
